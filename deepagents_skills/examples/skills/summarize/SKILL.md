# 文本总结技能

这个技能帮助将长文本内容进行结构化总结，提取关键信息，生成易于理解的摘要。

## 何时使用此技能

当你需要：
- 总结长篇文章或文档
- 提取文本的关键点
- 创建会议记录摘要
- 生成研究报告的执行摘要

## 总结流程

### 第一步：理解内容

1. **快速浏览** - 了解文本的整体结构和主题
2. **识别类型** - 确定是技术文档、新闻、研究报告还是其他类型
3. **确定目标** - 明确总结的目的和受众

### 第二步：提取信息

按照重要性提取以下信息：

1. **核心主题** - 文本主要讨论什么？
2. **关键论点** - 主要的观点或发现是什么？
3. **支撑证据** - 有哪些重要的数据或例子？
4. **结论建议** - 作者的结论或建议是什么？

### 第三步：生成总结

**总结格式模板：**

```markdown
## 总结

### 一句话概述
[用一句话概括核心内容]

### 关键要点
1. [要点一]
2. [要点二]
3. [要点三]

### 详细摘要
[2-3 段落的详细总结，保留重要细节]

### 重要数据/引用
- [关键数据点或引用]

### 行动建议（如适用）
- [基于内容的建议行动]
```

## 不同类型内容的总结策略

### 技术文档
- 强调功能和使用方法
- 保留重要的技术细节
- 列出关键 API 或配置

### 研究报告
- 强调研究方法和发现
- 保留关键数据和统计
- 突出结论和影响

### 新闻文章
- 使用 5W1H 框架（谁、什么、何时、何地、为什么、如何）
- 强调时效性信息
- 区分事实和观点

### 会议记录
- 列出决定和行动项
- 明确责任人和截止日期
- 突出重要讨论点

## 总结质量检查

### 检查清单
- [ ] 是否涵盖了所有关键点？
- [ ] 是否保持了客观性？
- [ ] 长度是否适合目标受众？
- [ ] 是否易于快速理解？
- [ ] 是否保留了必要的细节？

## 最佳实践

- **保持客观** - 不添加个人观点
- **使用原文语言** - 保留重要术语
- **结构清晰** - 使用标题和列表
- **长度适中** - 通常是原文的 10-25%
- **突出重点** - 把最重要的放在最前面
